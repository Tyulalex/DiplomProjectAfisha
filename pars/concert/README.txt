===============================================================================
                            pars/concert
===============================================================================


  Модуль concert_main_functions содержит функции необходимы для парсинга сайта concert.ru.
Для получения файла с url выбранного события необходимо вызвать функцию concert_main_functions.get_events_url_file(action_type_id).
Аргумент функции число, соответствующее виду события:
                      1 - concert
                      2 - theatre
На выходе будет создан файл иеющий следующее названия:
                      action_type_id == 1 - concert_urls.txt
                      action_type_id == 2 - theatre_urls.txt

Функции парсинга одинаковы для разных типов событий.

При запуске функции concert_main_functions.fetch_all_events_info(urls_file)
будет считываться каждый url в выбранном файле urls_file.
 Для каждого из url будет применена функция concert_main_functions.fetch_event_info(bs).
Эта функция возвращает словарь с ключами:
                      'event title' - название,
                      'event date' - дата,
                      'event weekday' - день недели,
                      'age limit' - возрастное ограничение,
                      'place' - место проведения,
                      'adress' - адресс места,
                      'genre' - жанр,
                      'description' - описание
После парсинга всех url из файла на выходе будет список словарей.


===============================================================================
                                  запуск
===============================================================================
Создаётся файл с импортом BeautifulSoup, requests и модуля concert_main_functions.
Для получения файла со списком url вызывается команда:

concert_main_functions.get_events_url_file(action_type_id)

Обновление файла проводить раз в день или чаще.
Далее для получения информации о событиях вызывается функция:

concert_main_functions.get_events_url_file(action_type_id).
